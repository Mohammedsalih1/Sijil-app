إضافة ميزة قراءة الإشعارات تلقائياً باستخدام Gemini API

أريد إضافة ميزة جديدة إلى تطبيق سِجِل دون التأثير على أي ميزة حالية أو تغيير منطق حفظ العمليات الموجود حالياً.

المتطلبات

عند ضغط المستخدم على زر إضافة عملية لا أريد أن يظهر الفورم مباشرة كما هو الآن، بل أريد أن تظهر نافذة تحتوي على:

اختيار نوع الإشعار

قائمة لاختيار نوع الإشعار:

بنكك
فوري
أوكاش

ويتم حفظ النوع المختار مع العملية عند إضافتها.

إضافة زرين جديدين

بعد اختيار نوع الإشعار تظهر زرين:

1. تصوير الإشعار

زر يفتح كاميرا الهاتف مباشرة لالتقاط صورة للإشعار المالي.

2. رفع من الهاتف

زر يسمح للمستخدم باختيار صورة إشعار موجودة مسبقاً في الهاتف.

تحليل الإشعار

لا أريد استخدام OCR التقليدي.

أريد استخدام Gemini API (أو أي حل Vision أقوى وأكثر دقة إذا كان متاحاً) لتحليل صورة الإشعار واستخراج البيانات المطلوبة بدقة عالية.

يجب أن يعمل مع الإشعارات العربية بصورة ممتازة.

البيانات المطلوب استخراجها
أولاً: المبلغ

استخراج مبلغ العملية من الإشعار.

مثال:

تم تحويل 5,000 جنيه

يتم استخراج:

{
  "amount": 5000
}
ثانياً: رقم الحساب

استخراج رقم الحساب الموجود بعد عبارة:

من حساب

أو ما يعادلها داخل الإشعار.

مثال:

من حساب 123456789

يتم استخراج:

{
  "accountNumber": "123456789"
}
ثالثاً: رقم العملية

إذا كان الإشعار من:

بنكك

استخراج رقم العملية ثم حفظ آخر 4 أرقام فقط.

مثال:

رقم العملية: 123456789

يتم حفظ:

6789
فوري

استخراج الرقم المرجعي ثم حفظ آخر 4 أرقام فقط.

مثال:

الرقم المرجعي: 987654321

يتم حفظ:

4321
أوكاش

استخراج رقم العملية أو الرقم المرجعي الموجود بالإشعار ثم حفظ آخر 4 أرقام فقط.

طريقة العمل

بعد رفع أو تصوير الإشعار:

إرسال الصورة إلى Gemini API.
استخراج البيانات المطلوبة.
تعبئة الفورم تلقائياً.
عرض البيانات للمستخدم للمراجعة والتأكد.

الفورم يجب أن يحتوي على:

رقم العملية (آخر 4 أرقام)
المبلغ
رقم الحساب
نوع الإشعار

ويتم تعبئتها تلقائياً.

تأكيد المستخدم

بعد مراجعة البيانات:

يضغط المستخدم على:

إضافة العملية
حفظ العملية

يجب استخدام نفس منطق الحفظ الحالي الموجود في التطبيق دون أي تعديل.

أي أن العملية الجديدة تحفظ بنفس الطريقة الحالية تماماً داخل Local Storage.

مهم جداً
لا تعدل أي ميزة موجودة حالياً.
لا تعدل نظام الإحصائيات.
لا تعدل نظام التفعيل والاشتراك.
لا تعدل التصدير PDF.
لا تعدل التصدير Excel.
لا تعدل البحث أو الفلاتر.
لا تعدل تصميم التطبيق الحالي.

أضف فقط ميزة قراءة الإشعار وتعبئة الفورم تلقائياً ثم حفظ العملية باستخدام النظام الحالي.

دقة الاستخراج

يجب إعطاء Gemini Prompt واضح وصارم لاستخراج البيانات فقط وإرجاع JSON منظم بالشكل التالي:

{
  "transactionLast4": "6789",
  "amount": "5000",
  "accountNumber": "123456789",
  "notificationType": "bankak"
}

ويجب تجاهل أي نصوص أخرى غير مطلوبة والتركيز فقط على:

آخر 4 أرقام من رقم العملية أو الرقم المرجعي.
المبلغ.
رقم الحساب المشار إليه بـ "من حساب".
نوع الإشعار المختار من المستخدم.

يجب أن تكون التجربة سريعة وسلسة وتعمل على الجوال بصورة ممتازة.
